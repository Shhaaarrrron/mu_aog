# 求援与换件交叉分析

## 功能概述

对求援记录与换件记录进行交叉匹配分析，建立求援件与实际换件之间的对应关系。

## 数据输入

| 文件 | 说明 |
|------|------|
| `data/raw/mu_aog.csv` | 求援记录数据 |
| `data/raw/21章拆换记录.csv` | 拆换记录数据 |

## 处理流程

### 1. 数据加载
读取求援记录和换件记录两个数据源，统计记录条数。

### 2. 数据预处理
- 日期字段统一转换为字符串格式
- 筛选有效安装记录（件号不为空）
- 初始化匹配状态标记

### 3. 核心匹配逻辑

对于每条求援记录，按照以下条件匹配换件记录：

| 匹配条件 | 说明 |
|----------|------|
| 件号匹配 | `MATNR == ZUMATNR` |
| 时间窗口 | 拆换日期 ≥ 求援日期 |
| 可用性 | 换件记录未被其他求援匹配 |
| 还回约束 | 若存在还回日期，则拆换日期 ≤ 还回日期 |

匹配策略：
- 优先使用最早可用的换件记录
- 每条换件记录仅能被匹配一次
- 未指定还回日期时，仅要求拆换日期在求援日期之后

### 4. 结果输出

输出文件：`data/processed/aog_with_exchange_match.csv`

统计维度：
- 总体匹配率
- 按优先级（ADPRI）分组统计
- 匹配记录样例展示

## 字段说明

### 求援记录关键字段
| 字段 | 说明 |
|------|------|
| MATNR | 件号 |
| ZZJRRQ | 求援日期 |
| ZZGHRQ | 还回日期 |
| ADPRI | 优先级 |
| EBELN | 订单号 |

### 换件记录关键字段
| 字段 | 说明 |
|------|------|
| ZUMATNR | 安装件号 |
| ZCHDAT | 拆换日期 |
| CHID | 拆换记录ID |

## 使用方式

```bash
python match_aog_exchange.py
```

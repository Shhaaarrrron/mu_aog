### 匹配逻辑

1. 遍历 21 章拆换记录的安装件（ZUMATNR）
2. 筛选件号相同且拆换日期（ZCHDAT）在求援日期（ZZJRRQ）之后的 AOG 记录
3. 如果 AOG 有归还日期（ZZGHRQ），则拆换日期需在归还日期之前
4. 匹配成功后，将拆换记录的 CHID 添加到 AOG 的 CHIDS 字段，剩余数量减 1
5. 当剩余数量为 0 时，该 AOG 不再参与后续匹配
6. 通过 `pd.merge` 将匹配的拆换记录完整信息合并到 AOG 记录中

### 业务说明

- **CHID 字段**：表示该求援件曾在哪个拆换操作中被安装到飞机上
- **剩余数量**：用于支持数量大于 1 的订单匹配多个拆换记录
- **PNR_ID**：用于标识该零件所属的 21 章章节，便于分类统计
- **拆换信息**：通过 CHID 关联 21 章拆换记录，包含飞机号、拆换日期、故障描述等详细信息

## 求援数据（含PNR_ID）

路径：`data/analysis/求援数据_with_pnr_id.csv`

业务描述：原始求援数据通过件号与 PNR_ID 映射表匹配后的结果，用于标识每个求援件所属的 21 章章节。

### 基本信息

| 属性       | 值                               |
| :--------- | :------------------------------- |
| 总记录数   | 2292 条                          |
| 唯一求援ID | 1721 个                          |
| 唯一件号数 | 109 个                           |
| 数据来源   | 求援数据.csv                     |
| 匹配率     | 100.0% (2292/2292)               |

### 字段说明

| 序号 | **字段名**   | 中文描述     | 数据来源     | 备注说明                     |
| :---: | :----------- | :----------- | :----------- | :--------------------------- |
| 1     | **id**       | 求援记录ID   | 求援数据     | 唯一标识每条求援记录         |
| 2     | **pn**       | 件号         | 求援数据     | 零件型号                     |
| 3     | **num**      | 数量         | 求援数据     | 求援数量                     |
| 4     | **company_name** | 航空公司 | 求援数据     | 发起求援的航空公司           |
| 5     | **aog_status** | 求援状态   | 求援数据     | Done/Confirm/Doing 等状态    |
| 6     | **create_time** | 创建时间  | 求援数据     | 求援记录创建时间             |
| 7     | **PNR_ID**   | 21章零件ID   | pn_id_mapper | 根据件号映射的 21 章章节号   |

### 匹配逻辑

1. 读取原始求援数据（`data/raw/求援数据.csv`）
2. 读取 PNR_ID 映射表（`data/raw/pn_id_mapper.csv`）
3. 通过件号（pn ↔ cur_pnr）进行左连接匹配
4. 将匹配到的 `matched_id` 重命名为 `PNR_ID`
5. 输出到 `data/analysis/求援数据_with_pnr_id.csv`

### 业务说明

- **PNR_ID**：用于标识该零件所属的 21 章章节，便于按章节分类统计求援件
- **匹配率 100%**：说明所有求援件号都能在 PNR_ID 映射表中找到对应关系
- **一对多关系**：同一个求援记录可能匹配到多个 PNR_ID（如件号 398908-5 同时匹配到 21-5X-35 和 21-6X-19）
